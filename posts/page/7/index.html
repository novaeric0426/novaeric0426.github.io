[OSTEP] Ch.19 - Paging - Faster Translations (TLBs)

Crux (핵심 문제) Paging은 모든 메모리 접근마다 page table을 조회해서 느리다. 어떻게 주소 변환을 빠르게 만드는가? 배경 & 동기 Ch.18의 문제: 모든 메모리 접근 = 실제 데이터 접근 + page table 조회 → 2배 느림. 해결책은 하드웨어 캐시 — TLB (Translation Lookaside Buffer): 최근 가상↔물리 주소 변환 결과를 캐싱하는 칩 내 하드웨어. TLB는 이름이 좀 이상하다(Translation Lookaside Buffer). 더 정확한 이름은 address-translation cache. 역사적 이유로 TLB라고 불림.
