FPS 개발일지 #08 — Chase AI: 감지/추적/복귀 상태 머신

패트롤 AI에 플레이어 감지, 추적, 복귀 상태를 추가한다. 히스테리시스로 상태 떨림을 방지하는 방법까지.
2026-02-21 · 4 min · Alan

이번 글에서 다룰 것#

패트롤만 하는 타겟은 맞고만 있을 뿐 위협이 되지 않는다. 플레이어를 감지하면 쫓아오고, 거리가 벌어지면 다시 패트롤로 돌아가는 Chase AI를 구현한다.

상태 머신 설계#

1Target_AI_State :: enum {
2    Idle,    // 고정 타겟 (웨이포인트 없음)
3    Patrol,  // 웨이포인트 패트롤
4    Chase,   // 플레이어 추적
5    Return,  // 패트롤 경로로 복귀
6}

4개의 상태와 전이 규칙은 다음과 같다:

1Idle (고정 타겟)
2  └─ 아무 것도 안 함
3
4Patrol ──[플레이어 감지]──→ Chase
5  ↑                            │
6  │                    [플레이어 이탈]
7  │                            ↓
8  └──[웨이포인트 도착]─── Return

Target 구조체 확장#

1Target :: struct {
2    // ... 기존 필드 ...
3
4    ai_state:      Target_AI_State,
5    detect_range:  f32,   // 감지 범위
6    lose_range:    f32,   // 추적 해제 범위
7    chase_speed:   f32,   // 추적 속도
8    return_target: Vec3,  // 복귀할 웨이포인트 위치
9}

생성 시 기본값:

1ai_state     = .Patrol,
2detect_range = 12.0,
3lose_range   = 18.0,
4chase_speed  = speed * 1.8,  // 패트롤보다 1.8배 빠름

히스테리시스 (Hysteresis)#

감지 범위와 해제 범위가 같으면 문제가 생긴다.

1detect_range = 15.0
2lose_range   = 15.0
3
4→ 플레이어가 15.0 부근에 서 있으면:
5  프레임1: 거리 14.99 → Chase
6  프레임2: 거리 15.01 → Return
7  프레임3: 거리 14.98 → Chase
8  ... (무한 반복)

이를 **상태 떨림(state flickering)**이라 한다. 해결 방법은 감지 범위와 해제 범위 사이에 간격을 두는 것이다.

1detect_range = 12.0  (이 안에 들어오면 Chase)
2lose_range   = 18.0  (이 밖으로 나가면 Return)

12~18 사이에 있으면 현재 상태가 유지된다. 이것이 히스테리시스다. 에어컨 온도 조절에서 “25도에 켜고 23도에 끈다"와 같은 원리다.

공통 이동 함수#

Patrol, Chase, Return 모두 “목표 지점을 향해 이동"하는 동작이 공통이다. 이것을 하나의 함수로 뽑았다.

 1move_target_toward :: proc(target: ^Target, goal: Vec3, speed: f32, world: ^World, dt: f32) -> f32 {
 2    diff := Vec3{goal.x - target.position.x, 0, goal.z - target.position.z}
 3    dist := linalg.length(diff)
 4    if dist < 0.01 do return dist
 5
 6    dir := diff / dist
 7    move_dist := speed * dt
 8
 9    // 축별 이동 + 충돌 롤백
10    target.position.x += dir.x * move_dist
11    if check_target_collision_world(target, target.position, world) {
12        target.position.x -= dir.x * move_dist
13    }
14    target.position.z += dir.z * move_dist
15    if check_target_collision_world(target, target.position, world) {
16        target.position.z -= dir.z * move_dist
17    }
18
19    return dist
20}

XZ 평면 이동, 월드 충돌, 남은 거리 반환까지 하나로 처리한다.

상태별 로직#

 1update_target :: proc(target: ^Target, world: ^World, player_pos: Vec3, dt: f32) {
 2    if !target.is_alive do return
 3
 4    player_diff := Vec3{player_pos.x - target.position.x, 0, player_pos.z - target.position.z}
 5    player_dist := linalg.length(player_diff)
 6
 7    switch target.ai_state {
 8    case .Idle:
 9        // 아무것도 안 함
10
11    case .Patrol:
12        // 감지 체크
13        if target.detect_range > 0 && player_dist < target.detect_range {
14            target.ai_state = .Chase
15            return
16        }
17        // 웨이포인트 이동 (기존 패트롤 로직)
18        // ...
19
20    case .Chase:
21        // 이탈 체크
22        if player_dist > target.lose_range {
23            nearest := find_nearest_waypoint(target)
24            target.current_waypoint = nearest
25            target.return_target = target.waypoints[nearest]
26            target.ai_state = .Return
27            return
28        }
29        // 플레이어를 향해 이동
30        move_target_toward(target, player_pos, target.chase_speed, world, dt)
31
32    case .Return:
33        // 복귀 중에도 감지 체크
34        if target.detect_range > 0 && player_dist < target.detect_range {
35            target.ai_state = .Chase
36            return
37        }
38        dist := move_target_toward(target, target.return_target, target.patrol_speed, world, dt)
39        if dist < PATROL_ARRIVE_THRESHOLD {
40            target.ai_state = .Patrol
41        }
42    }
43}

핵심 포인트:

  • Chase → Return: lose_range 밖으로 나가면 가장 가까운 웨이포인트를 찾아 복귀
  • Return → Patrol: 웨이포인트에 도착하면 패트롤 재개
  • Return → Chase: 복귀 중에도 플레이어가 다시 접근하면 바로 추적 전환

가장 가까운 웨이포인트 찾기#

 1find_nearest_waypoint :: proc(target: ^Target) -> int {
 2    best_idx := 0
 3    best_dist: f32 = max(f32)
 4    for wp, i in target.waypoints {
 5        diff := Vec3{wp.x - target.position.x, 0, wp.z - target.position.z}
 6        d := linalg.length(diff)
 7        if d < best_dist {
 8            best_dist = d
 9            best_idx = i
10        }
11    }
12    return best_idx
13}

Chase가 끝났을 때 타겟이 패트롤 경로에서 벗어나 있을 수 있다. 가장 가까운 웨이포인트로 복귀하면 자연스럽게 패트롤을 재개한다.

시각 인디케이터#

상태에 따라 머리 위 와이어 구 색이 바뀐다.

1indicator_color: rl.Color
2switch target.ai_state {
3case .Chase:         indicator_color = rl.RED
4case .Return:        indicator_color = rl.YELLOW
5case .Idle, .Patrol: indicator_color = rl.SKYBLUE
6}
7rl.DrawSphereWires(indicator_pos, 0.15, 4, 4, indicator_color)
  • 하늘색: 평화 (Idle/Patrol)
  • 빨강: 추적 중 (Chase)
  • 노랑: 복귀 중 (Return)

마치며#

이 글에서 구현한 것:

  • 4-상태 AI 상태 머신 (Idle/Patrol/Chase/Return)
  • 히스테리시스로 상태 떨림 방지 (detect=12, lose=18)
  • 공통 이동 함수 move_target_toward
  • 가장 가까운 웨이포인트로 복귀
  • 상태별 시각 인디케이터
