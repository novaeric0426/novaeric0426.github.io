FPS 개발일지 #18 — 서버 사이드 렉 보상

네트워크 지연 환경에서도 공정한 히트 판정을 위해 서버가 과거 시점으로 되감아 레이캐스트하는 렉 보상을 구현한다.
2026-03-15 · 5 min · Alan

이번 글에서 다룰 것#

PvP 전투를 구현했지만 한 가지 문제가 남아 있다. 네트워크 지연 때문에 클라이언트가 보는 화면과 서버의 실제 상태가 다르다. 클라이언트 화면에서는 정확히 머리에 조준했는데, 서버에서는 상대가 이미 다른 위치로 이동한 상태다. 결과: 미명중.

**렉 보상(Lag Compensation)**은 서버가 클라이언트의 시점으로 시간을 되감아 히트 판정을 하는 기법이다. Source 엔진(Counter-Strike, TF2)에서 사용하는 방식과 동일한 원리다.

문제 상황#

1시간 →  t=100       t=102       t=104
2
3서버:   적 위치 A → 적 위치 B → 적 위치 C (현재)
4
5클라이언트:         적 위치 A를 보고 사격 (t=102에 입력, 하지만 보고 있는 건 t=100의 상태)
6                    ↓
7서버 수신 (t=104):  현재 위치 C 기준으로 판정 → 미명중!

클라이언트는 항상 과거 상태를 보고 있다. 엔티티 보간 때문에 한 틱 뒤, 거기에 네트워크 지연까지 더해지면 수 틱 뒤의 세계를 보는 것이다. 클라이언트가 “여기 맞았다"고 보낸 시점에서 서버는 이미 다른 상태다.

해결: 서버가 되감는다#

1서버 수신 (t=104):
2  1. 클라이언트가 "나는 t=100 시점을 보고 쐈다" (client_view_tick=100)
3  2. 월드 히스토리에서 t=100 프레임 조회
4  3. 모든 엔티티를 t=100 위치로 되감기
5  4. 되감긴 위치로 레이캐스트 → 명중!
6  5. 위치 복원 (t=104로 되돌림)

클라이언트가 화면에서 본 그대로 판정한다. “내가 조준한 곳에 적이 있었으면 맞은 것이다.”

client_view_tick#

클라이언트가 “어느 시점을 보고 있는가"를 서버에 알려줘야 한다.

1Player_Input :: struct {
2    sequence:         u32,
3    client_view_tick: u32,   // 새로 추가 — 마지막으로 받은 스냅샷의 tick
4    mouse_delta:      Vec2,
5    // ...
6}

클라이언트는 스냅샷을 받을 때마다 틱 번호를 기록하고, 입력을 보낼 때 함께 전송한다.

1// 스냅샷 수신 시
2state.last_snapshot_tick = snapshot.tick
3
4// 입력 전송 시
5sent_input.client_view_tick = state.last_snapshot_tick

월드 히스토리#

서버는 매 틱의 엔티티 위치를 링 버퍼에 기록한다.

 1HISTORY_SIZE :: 64       // ~1초 (60Hz)
 2MAX_REWIND_TICKS :: 12   // 200ms 상한
 3
 4World_Frame :: struct {
 5    tick:              u32,
 6    player_positions:  [MAX_PLAYERS]Vec3,
 7    player_heights:    [MAX_PLAYERS]f32,
 8    player_is_active:  [MAX_PLAYERS]bool,
 9    player_is_alive:   [MAX_PLAYERS]bool,
10    target_positions:  [MAX_TARGETS]Vec3,
11    target_is_alive:   [MAX_TARGETS]bool,
12    target_count:      int,
13}
14
15World_History :: struct {
16    frames: [HISTORY_SIZE]World_Frame,
17    head:   int,
18    count:  int,
19}

64프레임 × 16.67ms ≈ 1초치를 보관한다. 대부분의 네트워크 환경에서 충분하다.

레이캐스트에 필요한 것은 위치뿐이다. HP, 속도, 상태 등은 저장하지 않는다.

기록#

서버의 simulate_tick 직후에 현재 프레임을 기록한다.

1// 서버 틱 루프
2simulate_tick(&server.game, &inputs, &server.history)
3record_world_frame(&server.history, &server.game, server.game.tick)

 1record_world_frame :: proc(history: ^World_History, game: ^Game_State, tick: u32) {
 2    frame := &history.frames[history.head]
 3    frame.tick = tick
 4
 5    for i := 0; i < MAX_PLAYERS; i += 1 {
 6        frame.player_positions[i] = game.players[i].position
 7        frame.player_heights[i] = game.players[i].current_height
 8        frame.player_is_active[i] = game.players[i].is_active
 9        frame.player_is_alive[i] = game.players[i].is_alive
10    }
11
12    count := min(len(game.targets), MAX_TARGETS)
13    frame.target_count = count
14    for i := 0; i < count; i += 1 {
15        frame.target_positions[i] = game.targets[i].position
16        frame.target_is_alive[i] = game.targets[i].is_alive
17    }
18
19    history.head = (history.head + 1) % HISTORY_SIZE
20    if history.count < HISTORY_SIZE {
21        history.count += 1
22    }
23}

되감기와 복원#

레이캐스트 전에 위치를 되감고, 끝나면 복원하는 save/restore 패턴을 사용한다.

 1save_positions :: proc(game: ^Game_State) -> Saved_Positions {
 2    saved: Saved_Positions
 3    for i := 0; i < MAX_PLAYERS; i += 1 {
 4        saved.player_positions[i] = game.players[i].position
 5        saved.player_heights[i] = game.players[i].current_height
 6    }
 7    count := min(len(game.targets), MAX_TARGETS)
 8    for i := 0; i < count; i += 1 {
 9        saved.target_positions[i] = game.targets[i].position
10    }
11    return saved
12}
13
14rewind_positions :: proc(game: ^Game_State, frame: ^World_Frame, shooter_index: int) {
15    for i := 0; i < MAX_PLAYERS; i += 1 {
16        if i == shooter_index do continue  // 사격자 본인은 되감지 않음
17        game.players[i].position = frame.player_positions[i]
18        game.players[i].current_height = frame.player_heights[i]
19    }
20    count := min(len(game.targets), frame.target_count)
21    for i := 0; i < count; i += 1 {
22        game.targets[i].position = frame.target_positions[i]
23    }
24}
25
26restore_positions :: proc(game: ^Game_State, saved: ^Saved_Positions) {
27    // save_positions의 역순
28    // ...
29}

shooter_index는 되감기에서 제외한다. 자기 자신의 위치는 현재 시점 그대로여야 한다 — 레이의 시작점이기 때문이다.

simulate_tick 통합#

 1simulate_tick :: proc(game: ^Game_State, inputs: ^[MAX_PLAYERS]Player_Input, history: ^World_History = nil) {
 2    // ...
 3    for i := 0; i < MAX_PLAYERS; i += 1 {
 4        // ...
 5        update_player(&game.players[i], &inputs[i], &game.world, TICK_DT)
 6
 7        // 렉 보상: 사격 중이고 client_view_tick이 있으면 되감기
 8        rewound := false
 9        saved: Saved_Positions
10        if history != nil && inputs[i].fire && inputs[i].client_view_tick > 0 {
11            rewind_ticks := calc_rewind_ticks(game.tick, inputs[i].client_view_tick)
12            if rewind_ticks > 0 {
13                frame := get_world_frame(history, inputs[i].client_view_tick)
14                if frame != nil {
15                    saved = save_positions(game)
16                    rewind_positions(game, frame, i)
17                    rewound = true
18                }
19            }
20        }
21
22        // 레이캐스트 (되감긴 위치 기준)
23        multi_result := update_gun(&game.guns[i], &inputs[i], &game.players[i],
24                                   game.targets[:], game.players[:], i, &game.world, TICK_DT)
25
26        // 복원
27        if rewound {
28            restore_positions(game, &saved)
29        }
30
31        // 히트 처리 ...
32    }
33}

핵심 흐름: 이동 → (되감기) → 사격 → (복원) → 히트 처리. 되감기는 사격 입력이 있을 때만 발생하고, 레이캐스트가 끝나면 즉시 복원한다.

history 파라미터는 옵션이다. 로컬 모드에서는 nil을 넘기면 되감기가 발생하지 않는다.

되감기 상한#

1MAX_REWIND_TICKS :: 12   // 200ms
2
3calc_rewind_ticks :: proc(current_tick: u32, client_view_tick: u32) -> u32 {
4    if client_view_tick == 0 || client_view_tick >= current_tick do return 0
5    diff := current_tick - client_view_tick
6    return min(diff, MAX_REWIND_TICKS)
7}

12틱 × 16.67ms ≈ 200ms. 이 이상 되감는 것은 허용하지 않는다. 핑이 200ms를 넘는 플레이어는 그만큼 불이익을 받지만, 핑이 낮은 플레이어가 과거로 너무 많이 돌아간 히트에 맞는 것을 방지한다.

되감기 않는 것들#

위치만 되감고, HP와 alive 상태는 현재 시점을 유지한다. 이미 죽은 플레이어를 과거로 되감아 다시 맞추는 것을 방지하기 위해서다. 사격 시점에는 살아있었더라도 현재 시점에서 이미 죽었으면 미명중 처리된다.

마치며#

이 글에서 구현한 것:

  • Player_Input.client_view_tick — 클라이언트가 보고 있던 시점
  • World_History — 64프레임 링 버퍼, 매 틱 위치 기록
  • rewind_positions / restore_positions — save/restore 패턴
  • simulate_tick 안에서 사격 시 되감기 → 레이캐스트 → 복원
  • MAX_REWIND_TICKS = 12 (200ms 상한)
  • 사격자 본인은 되감기 제외, HP/alive는 현재 시점 유지

이것으로 진짜 FPS 개발일지 시리즈가 마무리된다. 렉 보상까지 갖추면 네트워크 지연이 있어도 “내가 조준한 곳에 맞는다"는 직관이 유지된다. 클라이언트 예측이 이동의 즉각성을 보장하고, 렉 보상이 사격의 공정성을 보장한다 — 멀티플레이어 FPS의 두 기둥이 모두 완성되었다.
