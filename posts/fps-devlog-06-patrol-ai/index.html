FPS 개발일지 #06 — AI 타겟 패트롤 시스템

웨이포인트 기반 핑퐁 패트롤 AI 구현. 고정된 더미 타겟에서 움직이는 타겟으로.
2026-02-15 · 3 min · Alan

문제#

지금까지 타겟은 제자리에 서 있는 더미(dummy)였다. 사격 연습이 되긴 하지만 재미가 없다. 웨이포인트를 따라 이동하는 패트롤 AI를 구현한다.

Target 구조체 확장#

 1Target :: struct {
 2    position: Vec3,
 3    size:     Vec3,
 4    hp:       f32,
 5    max_hp:   f32,
 6    is_alive: bool,
 7    color:    rl.Color,
 8    // 패트롤 필드
 9    waypoints:        []Vec3,
10    current_waypoint: int,
11    patrol_speed:     f32,
12    patrol_direction: i32,  // +1: 정방향, -1: 역방향 (핑퐁)
13}

waypoints가 비어 있으면 기존처럼 고정 타겟으로 동작한다. 패트롤 타겟과 고정 타겟이 같은 타입으로 공존할 수 있다.

패트롤 타겟 생성#

 1create_patrol_target :: proc(
 2    waypoints: []Vec3,
 3    size: Vec3 = {1, 2, 1},
 4    hp: f32 = 100.0,
 5    color: rl.Color = rl.RED,
 6    speed: f32 = 2.5,
 7) -> Target {
 8    start_pos := waypoints[0] if len(waypoints) > 0 else Vec3{}
 9    return Target{
10        position         = start_pos,
11        size             = size,
12        hp               = hp,
13        max_hp           = hp,
14        is_alive         = true,
15        color            = color,
16        waypoints        = waypoints,
17        current_waypoint = 1 if len(waypoints) > 1 else 0,
18        patrol_speed     = speed,
19        patrol_direction = 1,
20    }
21}

첫 번째 웨이포인트에서 시작하고, 두 번째 웨이포인트를 향해 이동한다.

핑퐁 패트롤#

웨이포인트 끝에 도달하면 방향을 반대로 바꾸는 핑퐁(ping-pong) 방식이다.

1[0] → [1] → [2] → [1] → [0] → [1] → ...

 1PATROL_ARRIVE_THRESHOLD :: 0.2
 2
 3update_target :: proc(target: ^Target, world: ^World, dt: f32) {
 4    if len(target.waypoints) == 0 || !target.is_alive do return
 5
 6    goal := target.waypoints[target.current_waypoint]
 7    diff := Vec3{goal.x - target.position.x, 0, goal.z - target.position.z}
 8    dist := linalg.length(diff)
 9
10    if dist < PATROL_ARRIVE_THRESHOLD {
11        // 다음 웨이포인트로 (핑퐁)
12        next := target.current_waypoint + int(target.patrol_direction)
13        if next >= len(target.waypoints) {
14            target.patrol_direction = -1
15            next = target.current_waypoint - 1
16        } else if next < 0 {
17            target.patrol_direction = 1
18            next = target.current_waypoint + 1
19        }
20        target.current_waypoint = next
21        return
22    }
23
24    dir := diff / dist
25    move_dist := target.patrol_speed * dt
26    // 축별 이동 + 충돌 체크 (아래에서 설명)
27}

이동은 XZ 평면에서만 한다. Y 성분을 무시해서 높이 변화 없이 수평으로만 움직인다.

타겟의 충돌 처리#

타겟도 벽을 통과하면 안 된다. 플레이어와 같은 축별 이동 + 롤백 방식을 사용하되, 약간 작은 AABB(epsilon 축소)를 사용한다.

 1check_target_collision_world :: proc(target: ^Target, pos: Vec3, world: ^World) -> bool {
 2    EPSILON :: 0.01
 3    half := Vec3{
 4        target.size.x / 2 - EPSILON,
 5        target.size.y / 2 - EPSILON,
 6        target.size.z / 2 - EPSILON,
 7    }
 8    bbox := rl.BoundingBox{
 9        min = pos - half,
10        max = pos + half,
11    }
12    for &wall in world.walls {
13        if rl.CheckCollisionBoxes(bbox, get_wall_bbox(&wall)) do return true
14    }
15    for &obs in world.obstacles {
16        if rl.CheckCollisionBoxes(bbox, get_obstacle_bbox(&obs)) do return true
17    }
18    return false
19}

EPSILON으로 AABB를 살짝 줄인 이유는 벽에 정확히 맞닿았을 때 부동소수점 오차로 충돌이 감지되는 것을 방지하기 위해서다.

시각적 인디케이터#

패트롤 타겟 머리 위에 와이어 구를 표시해서 고정 타겟과 구분한다.

1if len(target.waypoints) > 0 {
2    indicator_pos := rl.Vector3{
3        target.position.x,
4        target.position.y + target.size.y / 2 + 0.7,
5        target.position.z,
6    }
7    rl.DrawSphereWires(indicator_pos, 0.15, 4, 4, rl.SKYBLUE)

마치며#

이 글에서 구현한 것:

  • 웨이포인트 기반 핑퐁 패트롤
  • XZ 평면 이동 (Y축 무시)
  • 타겟 월드 충돌 처리 (epsilon AABB)
  • 패트롤 인디케이터

고정 타겟과 패트롤 타겟이 같은 Target 구조체를 공유하므로, 기존 코드(데미지, 히트스캔 등)를 수정할 필요가 없다.

다음 글에서는 무기 전환 시스템(권총/소총/샷건)을 다룬다.

이 시리즈의 전체 소스 코드는 GitHub에서 확인할 수 있습니다.
