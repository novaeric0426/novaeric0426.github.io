FPS 개발일지 #14 — 멀티플레이어 플레이어 슬롯

단일 플레이어 구조를 MAX_PLAYERS 배열로 확장한다. 접속/해제 시 슬롯 할당, 다른 플레이어 렌더링까지.
2026-03-02 · 5 min · Alan

이번 글에서 다룰 것#

이전 글에서 구현한 서버는 첫 번째 접속자의 입력만 처리했다. 이번에는 최대 4명이 동시에 접속할 수 있도록 확장한다. 각 클라이언트에 플레이어 슬롯을 할당하고, 다른 플레이어를 캡슐로 렌더링한다.

Game_State 구조 변경#

 1// Before: 단일 플레이어
 2Game_State :: struct {
 3    player: Player,
 4    gun:    Gun,
 5    // ...
 6}
 7
 8// After: MAX_PLAYERS 배열
 9Game_State :: struct {
10    players:      [MAX_PLAYERS]Player,
11    guns:         [MAX_PLAYERS]Gun,
12    player_count: int,
13    // ...
14}

[dynamic]이 아닌 고정 크기 배열을 사용한다. 최대 4명이고, Game_Snapshot의 [MAX_PLAYERS] 배열과 1:1 대응되어야 하기 때문이다.

멀티플레이어 상수#

 1MAX_PLAYERS :: 4
 2
 3SPAWN_POINTS := [MAX_PLAYERS]Vec3{
 4    {0, 2, 10},
 5    {0, 2, -10},
 6    {10, 2, 0},
 7    {-10, 2, 0},
 8}
 9
10PLAYER_COLORS := [MAX_PLAYERS]rl.Color{
11    rl.BLUE,
12    rl.RED,
13    rl.GREEN,
14    rl.YELLOW,
15}

각 슬롯에 고유한 스폰 위치와 색상을 부여한다.

is_active 플래그#

1Player :: struct {
2    is_active: bool,  // 이 슬롯이 사용 중인가
3    // ...
4}

빈 슬롯과 사용 중인 슬롯을 구분한다. simulate_tick에서 비활성 슬롯은 건너뛴다.

서버: 접속 시 슬롯 할당#

 1Client_Info :: struct {
 2    endpoint:     net.Endpoint,
 3    id:           u32,
 4    connected:    bool,
 5    last_input:   Player_Input,
 6    sequence:     u32,
 7    player_index: int,  // Game_State.players 인덱스
 8}
 9
10server_handle_connect :: proc(server: ^Server_State, from: net.Endpoint) {
11    // 빈 슬롯 찾기
12    for &client, i in server.clients {
13        if !client.connected {
14            client.connected = true
15            client.endpoint = from
16            client.id = u32(i + 1)
17            client.player_index = i
18
19            // 이 슬롯에 플레이어/총기 생성
20            server.game.players[i] = create_player(SPAWN_POINTS[i])
21            server.game.guns[i] = create_gun_headless()
22            server.game.player_count += 1
23
24            send_packet(server.socket, from, .Connect, 0, client.id, nil)
25            return
26        }
27    }
28    // 빈 슬롯 없으면 거부
29}

client_id는 1부터 시작한다 (0은 “미접속” 상태). player_index는 0부터. 클라이언트 ID와 슬롯 인덱스의 관계는 player_index = client_id - 1.

서버: 해제 시 슬롯 비활성화#

 1server_handle_disconnect :: proc(server: ^Server_State, client_id: u32) {
 2    for &client in server.clients {
 3        if client.connected && client.id == client_id {
 4            server.game.players[client.player_index].is_active = false
 5            server.game.player_count -= 1
 6            client.connected = false
 7            client.id = 0
 8            return
 9        }
10    }
11}

플레이어를 삭제하지 않고 is_active = false로 비활성화한다. 배열에서 제거하면 인덱스가 밀리기 때문이다.

simulate_tick 확장#

 1simulate_tick :: proc(game: ^Game_State, inputs: ^[MAX_PLAYERS]Player_Input) {
 2    game.tick += 1
 3
 4    // 모든 활성 플레이어의 사망 타이머 갱신
 5    for i := 0; i < MAX_PLAYERS; i += 1 {
 6        if !game.players[i].is_active do continue
 7        update_player_death(&game.players[i], TICK_DT)
 8    }
 9
10    // AI: 가장 가까운 활성 플레이어 추적
11    update_targets_multiplayer(game.targets[:], &game.world, game.players[:], TICK_DT)
12
13    // 모든 활성+생존 플레이어 시뮬레이션
14    for i := 0; i < MAX_PLAYERS; i += 1 {
15        if !game.players[i].is_active do continue
16        if !game.players[i].is_alive do continue
17
18        update_player(&game.players[i], &inputs[i], &game.world, TICK_DT)
19        multi_result := update_gun(&game.guns[i], &inputs[i], &game.players[i],
20                                   game.targets[:], &game.world, TICK_DT)
21        // 히트 처리 ...
22    }
23}

시그니처가 ^Player_Input에서 ^[MAX_PLAYERS]Player_Input으로 바뀌었다. 서버는 각 클라이언트의 입력을 해당 슬롯 인덱스에 배치한다.

AI: 가장 가까운 플레이어 추적#

 1update_targets_multiplayer :: proc(targets: []Target, world: ^World, players: []Player, dt: f32) {
 2    for &target in targets {
 3        if !target.is_alive do continue
 4
 5        // 가장 가까운 활성 플레이어 찾기
 6        best_pos := Vec3{}
 7        best_dist: f32 = max(f32)
 8        found := false
 9
10        for &p in players {
11            if !p.is_active || !p.is_alive do continue
12            diff := Vec3{p.position.x - target.position.x, 0, p.position.z - target.position.z}
13            d := linalg.length(diff)
14            if d < best_dist {
15                best_dist = d
16                best_pos = p.position
17                found = true
18            }
19        }
20
21        if found {
22            update_target(&target, world, best_pos, dt)
23        } else {
24            update_target(&target, world, Vec3{99999, 0, 99999}, dt)
25        }
26    }
27}

싱글플레이어에서는 player_pos 하나를 받았지만, 멀티에서는 모든 활성 플레이어를 순회해서 가장 가까운 플레이어를 찾는다.

클라이언트: 다른 플레이어 렌더링#

클라이언트는 자기 슬롯을 1인칭 카메라로 보고, 다른 플레이어는 색이 칠해진 캡슐로 렌더링한다.

 1local_index := int(client_net.client_id) - 1
 2
 3// 자기 시점 카메라 사용
 4rl.BeginMode3D(game.players[local_index].camera)
 5
 6// 다른 플레이어를 캡슐로 렌더링
 7for i := 0; i < MAX_PLAYERS; i += 1 {
 8    if i == local_index do continue
 9    draw_other_player(&game.players[i], i)
10}

 1draw_other_player :: proc(player: ^Player, slot: int) {
 2    if !player.is_active || !player.is_alive do return
 3
 4    color := PLAYER_COLORS[slot % MAX_PLAYERS]
 5    pos := rl.Vector3{player.position.x, player.position.y, player.position.z}
 6
 7    // 몸통: 원기둥
 8    rl.DrawCylinder(pos, PLAYER_RADIUS, PLAYER_RADIUS, player.current_height * 0.8, 8, color)
 9    rl.DrawCylinderWires(pos, PLAYER_RADIUS, PLAYER_RADIUS, player.current_height * 0.8, 8, rl.BLACK)
10
11    // 머리: 구
12    head_pos := rl.Vector3{pos.x, pos.y + player.current_height * 0.8 + 0.2, pos.z}
13    rl.DrawSphere(head_pos, 0.25, color)
14    rl.DrawSphereWires(head_pos, 0.25, 6, 6, rl.BLACK)
15}

스냅샷: 모든 슬롯 전송#

1Game_Snapshot :: struct {
2    tick:         u32,
3    player_count: u8,
4    players:      [MAX_PLAYERS]Net_Player_State,
5    guns:         [MAX_PLAYERS]Net_Gun_State,
6    targets:      [MAX_TARGETS]Net_Target_State,
7    target_count: u8,
8    // ...
9}

4명분의 플레이어/총기 상태를 항상 포함한다. 비활성 슬롯은 is_active = false로 전송되므로 클라이언트가 알아서 무시한다.

함정: config/fov 초기화#

네트워크로 전송되지 않는 필드가 문제를 일으켰다. Player_Config(이동 속도, FOV 등)과 camera.fovy는 스냅샷에 포함되지 않는다. 로컬에서 create_player로 생성한 슬롯은 정상이지만, 원격 슬롯은 apply_player_state로만 초기화되므로 config가 제로 값이다.

camera.fovy = 0이면 3D 렌더링이 완전히 깨진다 — 하얀 화면만 보인다.

 1apply_player_state :: proc(player: ^Player, state: ^Net_Player_State) {
 2    // ... 필드 복사 ...
 3
 4    // config/camera가 초기화되지 않았으면 기본값 설정
 5    if player.config.fov == 0 {
 6        player.config = DEFAULT_PLAYER_CONFIG
 7    }
 8    if player.camera.fovy == 0 {
 9        player.camera.fovy = player.config.fov
10        player.camera.projection = .PERSPECTIVE
11        player.camera.up = {0, 1, 0}
12    }
13
14    sync_camera(player)
15}

교훈: 네트워크 상태에 포함되지 않는 필드는 수신 측에서 반드시 기본값을 보장해야 한다.

마치며#

이 글에서 구현한 것:

  • Game_State를 [MAX_PLAYERS] 배열 구조로 확장
  • is_active 플래그로 슬롯 활성화/비활성화
  • 서버: 접속 시 슬롯 할당 + 플레이어 생성, 해제 시 비활성화
  • simulate_tick이 모든 활성 플레이어를 독립적으로 시뮬레이션
  • update_targets_multiplayer: AI가 가장 가까운 플레이어를 추적
  • 클라이언트: local_index로 자기 시점 결정, 다른 플레이어는 캡슐 렌더링
  • 스냅샷에 4명분 상태 포함

이제 여러 명이 동시에 접속해서 같은 월드에서 뛰어다닐 수 있다. 하지만 입력이 서버를 거쳐야 하므로 자기 캐릭터도 느리게 반응한다. 다음 글에서 이 문제를 해결한다.

이 시리즈의 전체 소스 코드는 GitHub에서 확인할 수 있습니다.
