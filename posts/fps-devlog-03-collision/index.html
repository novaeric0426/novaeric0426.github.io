FPS 개발일지 #03 — AABB 슬라이드 콜리전

축별 이동과 롤백으로 구현하는 AABB 충돌 처리. 벽에 부딪혀도 미끄러지듯 움직이는 슬라이드 콜리전.
2026-02-06 · 4 min · Alan

문제#

이전 글까지는 플레이어가 벽과 장애물을 그냥 통과했다. y=0 바닥 체크만 있었을 뿐, 진짜 충돌 처리가 없었다.

이번 글에서는 AABB(축 정렬 바운딩 박스) 기반의 슬라이드 콜리전을 구현한다.

왜 AABB인가#

3D 충돌 처리에는 여러 방법이 있다:

  • 구(Sphere): 가장 간단하지만 벽 근처에서 부정확
  • AABB: 축 정렬 박스. 빠르고 구현이 간단
  • OBB/Convex Hull: 정확하지만 복잡

FPS에서 플레이어는 보통 직립 캡슐이나 박스로 근사한다. AABB면 충분하다.

플레이어 AABB#

1PLAYER_RADIUS :: 0.3
2
3get_player_bbox :: proc(position: Vec3, height: f32) -> rl.BoundingBox {
4    return rl.BoundingBox{
5        min = {position.x - PLAYER_RADIUS, position.y, position.z - PLAYER_RADIUS},
6        max = {position.x + PLAYER_RADIUS, position.y + height, position.z + PLAYER_RADIUS},
7    }
8}

플레이어의 위치(position.y)가 발 위치이고, 거기서 height만큼 위로 뻗는 박스다. 앉으면 height가 줄어들어 AABB도 작아진다.

월드 AABB 헬퍼#

벽과 장애물도 AABB로 표현한다.

 1get_wall_bbox :: proc(wall: ^Wall) -> rl.BoundingBox {
 2    half := Vec3{wall.size.x / 2, wall.size.y / 2, wall.size.z / 2}
 3    return rl.BoundingBox{
 4        min = wall.position - half,
 5        max = wall.position + half,
 6    }
 7}
 8
 9get_obstacle_bbox :: proc(obs: ^Obstacle) -> rl.BoundingBox {
10    switch obs.type {
11    case .Box, .Ramp:
12        half := Vec3{obs.size.x / 2, obs.size.y / 2, obs.size.z / 2}
13        return rl.BoundingBox{min = obs.position - half, max = obs.position + half}
14    case .Cylinder:
15        r := obs.size.x
16        h := obs.size.y
17        return rl.BoundingBox{
18            min = {obs.position.x - r, obs.position.y, obs.position.z - r},
19            max = {obs.position.x + r, obs.position.y + h, obs.position.z + r},
20        }
21    }
22    return rl.BoundingBox{}
23}

실린더도 AABB로 근사한다. 정확한 원통 충돌 판정보다 훨씬 간단하고, 게임플레이에 큰 차이가 없다.

슬라이드 콜리전#

핵심 아이디어는 축별(axis-by-axis) 이동이다.

11. X축으로 이동 → 충돌 체크 → 겹치면 X 롤백
22. Y축으로 이동 → 충돌 체크 → 겹치면 Y 롤백
33. Z축으로 이동 → 충돌 체크 → 겹치면 Z 롤백

이렇게 하면 벽에 대각선으로 부딪혀도 막히지 않는 축 방향으로는 미끄러지듯 이동한다. 이것이 “슬라이드” 콜리전이다.

 1apply_velocity :: proc(player: ^Player, world: ^World, dt: f32) {
 2    // X축
 3    player.position.x += player.velocity.x * dt
 4    if check_collision_world(get_player_bbox(player.position, player.current_height), world) {
 5        player.position.x -= player.velocity.x * dt
 6        player.velocity.x = 0
 7    }
 8
 9    // Y축
10    player.position.y += player.velocity.y * dt
11    if check_collision_world(get_player_bbox(player.position, player.current_height), world) {
12        player.position.y -= player.velocity.y * dt
13        player.velocity.y = 0
14    }
15
16    // Z축
17    player.position.z += player.velocity.z * dt
18    if check_collision_world(get_player_bbox(player.position, player.current_height), world) {
19        player.position.z -= player.velocity.z * dt
20        player.velocity.z = 0
21    }
22}

각 축에서 이동 후 겹침이 발생하면 해당 축만 롤백하고 속도를 0으로 만든다. 단순하지만 효과적이다.

월드 전체 충돌 체크#

1check_collision_world :: proc(bbox: rl.BoundingBox, world: ^World) -> bool {
2    for &wall in world.walls {
3        if rl.CheckCollisionBoxes(bbox, get_wall_bbox(&wall)) do return true
4    }
5    for &obs in world.obstacles {
6        if rl.CheckCollisionBoxes(bbox, get_obstacle_bbox(&obs)) do return true
7    }
8    return false
9}

Raylib의 CheckCollisionBoxes가 두 AABB의 겹침을 판정한다. 벽과 장애물 전체를 순회하면서 하나라도 겹치면 true를 반환한다.

장애물 위에 올라서기#

바닥만 y=0으로 체크하던 것을 장애물 위에도 올라설 수 있게 확장했다.

 1update_ground_check :: proc(player: ^Player, world: ^World) {
 2    ground_level: f32 = 0.0
 3
 4    // 플레이어 발 아래 얇은 박스로 바닥 탐지
 5    feet_probe := rl.BoundingBox{
 6        min = {player.position.x - PLAYER_RADIUS, player.position.y - 0.05, player.position.z - PLAYER_RADIUS},
 7        max = {player.position.x + PLAYER_RADIUS, player.position.y, player.position.z + PLAYER_RADIUS},
 8    }
 9
10    obstacle_ground: f32 = ground_level
11    for &obs in world.obstacles {
12        bbox := get_obstacle_bbox(&obs)
13        if rl.CheckCollisionBoxes(feet_probe, bbox) {
14            top := bbox.max.y
15            if top > obstacle_ground {
16                obstacle_ground = top
17            }
18        }
19    }
20
21    if player.position.y <= obstacle_ground {
22        player.position.y = obstacle_ground
23        player.velocity.y = 0
24        player.is_grounded = true
25    } else {
26        player.is_grounded = false
27    }
28}

발 바로 아래에 얇은 AABB(feet_probe)를 만들어서 장애물 윗면과 겹치는지 확인한다. 겹치는 장애물 중 가장 높은 윗면을 새 바닥으로 사용한다.

마치며#

이 글에서 구현한 것:

  • 플레이어/벽/장애물의 AABB 정의
  • 축별 이동 + 롤백 기반 슬라이드 콜리전
  • 장애물 위 착지 판정

이 AABB 헬퍼 함수들(get_wall_bbox, get_obstacle_bbox)은 나중에 히트스캔의 벽 차폐 판정에서도 재사용된다.
