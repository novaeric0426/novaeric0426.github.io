FPS 개발일지 #10 — 직렬화 가능한 게임 상태

네트워크 전송을 위해 게임 상태를 POD 구조체로 추출하고, mem.copy 기반 직렬화를 구현한다.
2026-02-27 · 4 min · Alan

이번 글에서 다룰 것#

멀티플레이어에서 서버는 매 틱마다 게임 상태를 클라이언트에 보내야 한다. 그런데 Player나 Gun 구조체에는 Raylib의 Camera3D, Model 같은 GPU 리소스가 포함되어 있다. 이런 필드는 네트워크로 보낼 수 없다.

이번 글에서는 네트워크 전송에 필요한 필드만 골라낸 POD(Plain Old Data) 구조체를 만들고, 게임 상태를 바이트 배열로 직렬화하는 시스템을 구현한다.

왜 POD 구조체가 필요한가#

 1Player 구조체:
 2  position: Vec3     ← 네트워크로 보내야 함
 3  velocity: Vec3     ← 네트워크로 보내야 함
 4  camera:   Camera3D ← GPU 리소스, 보낼 수 없음
 5  config:   Player_Config ← 양쪽 다 동일, 보낼 필요 없음
 6
 7Gun 구조체:
 8  model:    Model    ← GPU 리소스
 9  state:    Gun_State ← 보내야 함
10  weapon_ammo: [..]  ← 보내야 함

포인터, 핸들, GPU 리소스를 제거하고 순수한 값 타입만 남긴 구조체가 POD다. mem.copy로 바이트 배열에 복사할 수 있어야 한다.

카메라, 모델, 설정값(config) 등은 빠져 있다. 클라이언트가 스냅샷을 받은 뒤 sync_camera()로 카메라를 재구성하면 된다.

Game_Snapshot#

1MAX_TARGETS :: 16
2
3Game_Snapshot :: struct {
4    tick:         u32,
5    player:       Net_Player_State,
6    gun:          Net_Gun_State,
7    targets:      [MAX_TARGETS]Net_Target_State,
8    target_count: u8,
9}

고정 크기 배열을 사용한다. 동적 배열([dynamic])은 포인터를 포함하므로 mem.copy로 직렬화할 수 없다. target_count로 실제 유효한 타겟 수를 기록한다.

Extract / Apply 패턴#

게임 상태와 네트워크 상태 사이의 변환은 extract/apply 함수 쌍으로 처리한다.

Extract (Game → Net):

 1extract_player_state :: proc(player: ^Player) -> Net_Player_State {
 2    return Net_Player_State{
 3        position       = player.position,
 4        velocity       = player.velocity,
 5        yaw            = player.yaw,
 6        pitch          = player.pitch,
 7        move_state     = player.move_state,
 8        is_grounded    = player.is_grounded,
 9        current_height = player.current_height,
10        hp             = player.hp,
11        is_alive       = player.is_alive,
12        death_timer    = player.death_timer,
13    }
14}

Apply (Net → Game):

 1apply_player_state :: proc(player: ^Player, state: ^Net_Player_State) {
 2    player.position       = state.position
 3    player.velocity       = state.velocity
 4    player.yaw            = state.yaw
 5    player.pitch          = state.pitch
 6    player.move_state     = state.move_state
 7    player.is_grounded    = state.is_grounded
 8    player.current_height = state.current_height
 9    player.hp             = state.hp
10    player.is_alive       = state.is_alive
11    player.death_timer    = state.death_timer
12
13    // 카메라는 position/yaw/pitch에서 재구성
14    sync_camera(player)
15}

Gun, Target도 동일한 패턴으로 extract/apply 함수를 작성했다.

스냅샷 캡처와 적용#

 1capture_snapshot :: proc(game: ^Game_State, tick: u32) -> Game_Snapshot {
 2    snapshot: Game_Snapshot
 3    snapshot.tick = tick
 4    snapshot.player = extract_player_state(&game.player)
 5    snapshot.gun = extract_gun_state(&game.gun)
 6
 7    count := min(len(game.targets), MAX_TARGETS)
 8    snapshot.target_count = u8(count)
 9    for i := 0; i < count; i += 1 {
10        snapshot.targets[i] = extract_target_state(&game.targets[i])
11    }
12    return snapshot
13}
14
15apply_snapshot :: proc(game: ^Game_State, snapshot: ^Game_Snapshot) {
16    apply_player_state(&game.player, &snapshot.player)
17    apply_gun_state(&game.gun, &snapshot.gun)
18
19    count := min(int(snapshot.target_count), len(game.targets))
20    for i := 0; i < count; i += 1 {
21        apply_target_state(&game.targets[i], &snapshot.targets[i])
22    }
23}

바이너리 직렬화#

POD 구조체의 장점은 mem.copy 한 번으로 직렬화가 끝난다는 것이다.

 1SNAPSHOT_SIZE :: size_of(Game_Snapshot)
 2
 3serialize_snapshot :: proc(snapshot: ^Game_Snapshot, buf: []u8) -> int {
 4    if len(buf) < SNAPSHOT_SIZE do return 0
 5    mem.copy(raw_data(buf), snapshot, SNAPSHOT_SIZE)
 6    return SNAPSHOT_SIZE
 7}
 8
 9deserialize_snapshot :: proc(buf: []u8, snapshot: ^Game_Snapshot) -> bool {
10    if len(buf) < SNAPSHOT_SIZE do return false
11    mem.copy(snapshot, raw_data(buf), SNAPSHOT_SIZE)
12    return true
13}

별도의 직렬화 라이브러리 없이, 구조체 메모리를 그대로 복사한다. Odin의 구조체는 기본적으로 C와 동일한 메모리 레이아웃을 갖기 때문에 가능하다.

Player_Input도 같은 방식으로 직렬화한다. 클라이언트가 서버에 입력을 보낼 때 사용된다.

디버그 라운드트립 테스트#

직렬화가 제대로 되는지 확인하기 위해 L 키로 라운드트립 테스트를 추가했다.

 1debug_snapshot_round_trip :: proc(game: ^Game_State) {
 2    snapshot := capture_snapshot(game, 0)
 3
 4    buf: [SNAPSHOT_SIZE]u8
 5    written := serialize_snapshot(&snapshot, buf[:])
 6
 7    restored: Game_Snapshot
 8    deserialize_snapshot(buf[:], &restored)
 9
10    apply_snapshot(game, &restored)
11    fmt.printf("Snapshot round-trip OK, %d bytes\n", written)
12}

캡처 → 직렬화 → 역직렬화 → 적용. 게임이 정상적으로 돌아가면 성공이다. 눈에 보이는 변화가 없어야 올바른 것이다.

마치며#

이 글에서 구현한 것:

  • Net_Player_State, Net_Gun_State, Net_Target_State POD 구조체
  • Game_Snapshot — 고정 크기 배열, tick 번호 포함
  • Extract/Apply 함수 쌍으로 게임 상태 ↔ 네트워크 상태 변환
  • mem.copy 기반 바이너리 직렬화/역직렬화
  • 라운드트립 디버그 테스트

이제 게임 상태를 바이트 배열로 변환해서 네트워크로 보낼 준비가 되었다.
