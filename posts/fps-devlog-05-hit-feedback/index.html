FPS 개발일지 #05 — 히트마커와 데미지 넘버

히트마커 X 오버레이와 플로팅 데미지 넘버로 사격 피드백을 강화한다.
2026-02-12 · 3 min · Alan

문제#

총을 쏴서 타겟의 HP가 줄어들지만, 플레이어 입장에서 “맞았다"는 느낌이 전혀 없었다. 히트마커와 데미지 넘버를 추가한다.

Fire_Result#

히트 정보를 반환하도록 update_gun의 시그니처를 변경했다.

 1Fire_Result :: struct {
 2    hit:          bool,
 3    killed:       bool,
 4    hit_position: Vec3,
 5    damage:       f32,
 6}
 7
 8update_gun :: proc(gun: ^Gun, player: ^Player, targets: []Target, world: ^World, dt: f32) -> Fire_Result {
 9    // ...
10    result := fire_gun(gun, player, targets, world)
11    if result.hit {
12        gun.hitmarker_timer = HITMARKER_DURATION
13        gun.hitmarker_is_kill = result.killed
14    }
15    return result
16}

발사 함수가 히트 여부, 킬 여부, 히트 위치, 데미지를 담은 Fire_Result를 반환한다. 호출자는 이 결과를 보고 UI 피드백을 생성한다.

히트마커#

크로스헤어 위에 X자 형태의 히트마커를 0.15초간 표시한다.

 1HITMARKER_DURATION :: 0.15
 2
 3// draw_gun_ui 안에서
 4if gun.hitmarker_timer > 0 {
 5    alpha := u8(gun.hitmarker_timer / HITMARKER_DURATION * 255)
 6    color := gun.hitmarker_is_kill ? rl.Color{255, 0, 0, alpha} : rl.Color{255, 255, 255, alpha}
 7
 8    cx := i32(WINDOW_WIDTH / 2)
 9    cy := i32(WINDOW_HEIGHT / 2)
10    gap: i32 = 5
11    len: i32 = 10
12
13    // X 형태: 크로스헤어 주변 대각선 4개
14    rl.DrawLine(cx - gap, cy - gap, cx - gap - len, cy - gap - len, color)
15    rl.DrawLine(cx + gap, cy - gap, cx + gap + len, cy - gap - len, color)
16    rl.DrawLine(cx - gap, cy + gap, cx - gap - len, cy + gap + len, color)
17    rl.DrawLine(cx + gap, cy + gap, cx + gap + len, cy + gap + len, color)
18}

일반 히트는 흰색, 킬은 빨간색이다. 타이머가 줄어들면서 알파 값이 함께 감소해 자연스럽게 페이드아웃된다.

데미지 넘버#

맞은 위치에서 숫자가 떠오르며 사라지는 효과다.

1DAMAGE_NUMBER_DURATION   :: 0.8
2DAMAGE_NUMBER_RISE_SPEED :: 1.5
3
4Damage_Number :: struct {
5    position: Vec3,
6    amount:   f32,
7    lifetime: f32,
8}

생성#

1// 히트 시 데미지 넘버 생성
2if fire_result.hit {
3    append(&game.damage_numbers, Damage_Number{
4        position = fire_result.hit_position,
5        amount   = fire_result.damage,
6        lifetime = DAMAGE_NUMBER_DURATION,
7    })
8}

fire_result의 hit_position이 레이와 타겟 AABB의 교차점이다. 정확히 맞은 지점에서 숫자가 뜬다.

업데이트#

 1update_damage_numbers :: proc(numbers: ^[dynamic]Damage_Number, dt: f32) {
 2    i := 0
 3    for i < len(numbers) {
 4        numbers[i].position.y += DAMAGE_NUMBER_RISE_SPEED * dt
 5        numbers[i].lifetime -= dt
 6        if numbers[i].lifetime <= 0 {
 7            ordered_remove(numbers, i)
 8        } else {
 9            i += 1
10        }
11    }
12}

매 프레임 위로 떠오르고(position.y += ...), lifetime이 0이 되면 제거한다. ordered_remove로 순서를 유지하면서 삭제한다.

렌더링#

1draw_damage_numbers :: proc(numbers: []Damage_Number, camera: ^rl.Camera3D) {
2    for &dn in numbers {
3        alpha := u8(dn.lifetime / DAMAGE_NUMBER_DURATION * 255)
4        screen_pos := rl.GetWorldToScreen(rl.Vector3{dn.position.x, dn.position.y, dn.position.z}, camera^)
5
6        text := rl.TextFormat("%.0f", dn.amount)
7        rl.DrawText(text, i32(screen_pos.x), i32(screen_pos.y), 20, rl.Color{255, 255, 0, alpha})
8    }
9}

3D 위치를 GetWorldToScreen으로 화면 좌표로 변환한 뒤, 2D 텍스트로 그린다. 남은 lifetime에 비례해 페이드아웃된다.

마치며#

이 글에서 구현한 것:

  • Fire_Result 구조체로 히트 정보 반환
  • 히트마커 (흰색=히트, 빨간색=킬, 0.15초 페이드)
  • 데미지 넘버 (노란색, 떠오르며 0.8초 페이드)

사운드는 아직 없다. 시각 피드백만으로도 사격의 “느낌"이 크게 달라진다.
